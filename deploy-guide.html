5분 즉시 배포 (Railway 기준)

RECOMMENDED
GitHub 계정만 있으면 됩니다. 신용카드 불필요. 무료로 시작!
1
GitHub에 코드 올리기

아직 GitHub에 코드가 없다면 새 저장소 생성 후 업로드하세요

bash
# 프로젝트 폴더에서 실행
git init
git add .
git commit -m "초기 커밋: DANGUN 플랫폼"
git branch -M main

# GitHub 저장소 URL로 변경하세요
git remote add origin https://github.com/YOUR_USER/dangun-platform.git
git push -u origin main
2
Railway 계정 생성 및 로그인

GitHub 계정으로 소셜 로그인 → 바로 사용 가능

railway.app 열기
3
"New Project" → "Deploy from GitHub repo" 클릭

dangun-platform 저장소 선택 → 자동으로 railway.toml 감지

4
환경변수(Variables) 설정

Railway 프로젝트 → Variables 탭에서 아래 값 추가:

env
JWT_SECRET=무작위32자이상문자열로변경필수
ADMIN_ID=superadmin
ADMIN_PASSWORD=Admin1234!
ADMIN_NAME=최고관리자
ADMIN_EMAIL=admin@dangun.com
SCHEDULER_ENABLED=true
5
배포 완료!

Railway가 자동 빌드 후 URL 발급. "Settings → Domains"에서 커스텀 도메인 연결 가능

Railway 무료 플랜은 월 500시간 제한. 지속 운영 시 $5/월 플랜 권장

완전 무료 영구 배포 (Oracle Cloud)

ALWAYS FREE
1
Oracle Cloud 계정 생성

Always Free 티어: ARM 4코어 · 24GB RAM · 200GB 스토리지 (영구 무료)

Oracle Cloud 무료 가입
2
Ubuntu 22.04 VM 인스턴스 생성

Compute → Instances → Create Instance → VM.Standard.A1.Flex (ARM) 선택

3
코드 업로드 후 install.sh 실행
bash
# 코드 압축 후 서버에 업로드
scp -r ./dangun-platform ubuntu@서버IP:~

# SSH 접속
ssh ubuntu@서버IP

# 설치 스크립트 실행 (5~10분 소요)
cd dangun-platform
sudo bash install.sh
4
Oracle Cloud 보안 그룹에서 80, 443 포트 열기

Networking → Virtual Cloud Networks → Security Lists → Ingress Rules에 80, 443 추가

5
접속 완료!

http://서버IP/ 로 접속. 도메인 연결 후 SSL 인증서 발급 권장

Railway 배포 상세 가이드

railway.toml 파일이 이미 설정되어 있습니다. GitHub 연결만 하면 자동 배포됩니다.
Railway CLI로 배포 (터미널)
bash
# Railway CLI 설치
npm install -g @railway/cli

# Railway 로그인
railway login

# 프로젝트 초기화 (처음 한 번만)
railway init

# 환경변수 설정
railway variables set JWT_SECRET="your-64-char-secret"
railway variables set ADMIN_PASSWORD="Admin1234!"
railway variables set ADMIN_ID="superadmin"
railway variables set ADMIN_EMAIL="admin@dangun.com"
railway variables set SCHEDULER_ENABLED="true"

# 배포 실행
railway up

# 배포 URL 확인
railway domain

# 로그 실시간 확인
railway logs --follow
SQLite 영구 저장 설정

Railway는 배포 시 컨테이너가 재생성되므로 SQLite DB 파일 유지를 위해 Volume을 설정해야 합니다.

1
Railway 대시보드 → 프로젝트 → "Add Volume" 클릭
2
Mount Path: /app/database 설정
3
환경변수 추가: DB_PATH=/app/database/dangun.db

Render.com 배포 상세 가이드

render.yaml 파일이 이미 설정되어 있어 저장소 연결만으로 자동 배포됩니다.
1
render.com 가입 → GitHub 연결 render.com 열기
2
"New" → "Web Service" → GitHub 저장소 선택

render.yaml을 자동 감지하여 설정을 불러옵니다

3
Environment Variables에서 ADMIN_PASSWORD 설정

render.yaml에서 sync:false로 표시된 항목은 수동 입력 필요

4
"Create Web Service" 클릭 → 자동 배포

빌드 완료 후 *.onrender.com URL이 발급됩니다

Persistent Disk 설정 (유료)

Render 무료 플랜은 배포 시 파일시스템이 초기화됩니다. SQLite 영구 보존을 위해 Disk 추가 필요:

yaml (render.yaml 내)
    disk:
      name: dangun-db
      mountPath: /opt/render/project/src/database
      sizeGB: 1    # 월 $0.25/GB

VPS / 클라우드 서버 배포

원클릭 설치 스크립트
Ubuntu 20.04/22.04/24.04 LTS 지원. Node.js, PM2, Nginx 자동 설치!
bash
# 방법 1: 프로젝트 폴더에서 직접 실행
sudo bash install.sh

# 방법 2: Git clone 후 실행
git clone https://github.com/YOUR_USER/dangun-platform.git
cd dangun-platform
sudo bash install.sh
install.sh가 자동으로 처리하는 것: Node.js 설치 → 환경변수 생성 → npm install → PM2 설정 → Nginx 리버스 프록시 → 방화벽 설정 → 자동 시작 등록
SSL 인증서 설치 (HTTPS)
bash
# Certbot 설치 (Let's Encrypt 무료 SSL)
sudo apt install -y certbot python3-certbot-nginx

# SSL 인증서 발급 (도메인이 연결된 경우)
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 자동 갱신 테스트
sudo certbot renew --dry-run

# 메모리/CPU 모니터링

Docker 컨테이너 배포

기본 Docker Compose 실행
bash
# 1. 환경변수 파일 생성
cp .env.example .env

# 2. .env 파일 편집 (JWT_SECRET, ADMIN_PASSWORD 변경 필수)
nano .env

# 3. 빌드 및 백그라운드 실행
docker-compose up -d --build

# 4. 실행 상태 확인
docker-compose ps

# 5. 로그 확인
docker-compose logs -f app

# 6. 재시작
docker-compose restart app

# 7. 중지
docker-compose down
DB 백업 및 복원
bash
# DB 백업 (볼륨에서 로컬로)
docker cp dangun-app:/app/database/dangun.db ./backup_$(date +%Y%m%d).db

# DB 복원
docker cp ./backup_20250418.db dangun-app:/app/database/dangun.db
docker-compose restart app

# 볼륨 목록 확인
docker volume ls | grep dangun

# 테스트 데이터 생성
docker exec dangun-app node database/seed.js

# 즉시 지급 실행
docker exec dangun-app node scheduler.js --now
GitHub Actions + Docker Hub 자동 배포

.github/workflows/deploy.yml에 SSH 배포가 설정되어 있습니다.
GitHub Secrets에 아래 값을 추가하면 main 브랜치 push 시 자동 배포됩니다.

GitHub Secrets 설정 목록
SSH_HOST        = 서버 IP 또는 도메인
SSH_USER        = ubuntu (또는 dangun)
SSH_PRIVATE_KEY = SSH 개인키 내용 (-----BEGIN...)
SSH_PORT        = 22

GitHub 저장소 → Settings → Secrets and variables → Actions → New repository secret

환경변수 전체 목록

변수명 기본값 필수 설명
NODE_ENV production 선택 실행 환경 (production/development)
PORT 3000 선택 서버 포트 (Render: 10000)
DB_PATH /app/database/dangun.db 선택 SQLite DB 파일 경로
JWT_SECRET - 필수 JWT 서명키 (64자 무작위 문자열 권장)
JWT_EXPIRES_IN 7d 선택 JWT 만료 시간
ADMIN_ID superadmin 필수 최초 관리자 로그인 ID
ADMIN_PASSWORD Admin1234! 필수 관리자 초기 비밀번호 (반드시 변경!)
ADMIN_NAME 최고관리자 선택 관리자 표시 이름
ADMIN_EMAIL admin@dangun.com 선택 관리자 이메일
SCHEDULER_ENABLED true 선택 매주 금요일 자동 지급 스케줄러 활성화
LOG_LEVEL info 선택 로그 레벨 (error/warn/info/debug)
JWT_SECRET 안전하게 생성하기
bash
# Linux / macOS
openssl rand -base64 64

# Node.js
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# Windows PowerShell
[System.Convert]::ToBase64String([System.Security.Cryptography.RandomNumberGenerator]::GetBytes(64))

DB 자동 백업 설정 (cron)
bash
# crontab 편집
crontab -e

# 아래 줄 추가 (매일 새벽 3시 자동 백업)
0 3 * * * cp /opt/dangun/database/dangun.db /opt/dangun/database/backup_$(date +\%Y\%m\%d).db && find /opt/dangun/database -name "backup_*.db" -mtime +30 -delete

# 백업 목록 확인
ls -lh /opt/dangun/database/backup_*.db
